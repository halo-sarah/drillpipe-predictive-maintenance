About this model

Loss per job is computed step-by-step from stimulation chemistry (HCl, HF, NH₄Cl), inhibitor loading, and coating protection factor. A confidence penalty α inflates the loss estimate conservatively when ML model certainty is low.

DS-1 grades follow the API/ISO drill pipe inspection standard. The integrity score (0–100) combines thickness condition, spike anomaly, ML confidence, and RUL risk.

About PipeIntel

PipeIntel is a predictive maintenance system for coated drill pipe used in acid stimulation service. It combines ultrasonic waveform analysis, machine learning coating classification, and physics-based corrosion modeling to produce a single integrity score and maintenance recommendation per reading.

The system is designed as an internal engineering tool — data-first, minimal interface, explainable outputs. It is a portfolio project built to demonstrate the intersection of domain engineering knowledge and applied ML.

Integrity scoring

The integrity score (0–100) is a composite index. A thickness-based base score is reduced by up to three penalty terms reflecting anomaly detection, model confidence, and remaining life risk.

ComponentMax effect
Wall thickness condition0–100 (base)
Spike / anomaly penalty−20 pts
ML confidence deficit (α × deficit)−15 pts
RUL risk (< 20 jobs remaining)−15 pts
ScoreStatusRecommendation
≥ 70NORMALContinue operation
40–70WARNINGSchedule inspection
< 40CRITICALImmediate shutdown
Deployment guide

Clone the repository and install dependencies:

git clone https://github.com/halo-sarah/drillpipe-predictive-maintenance cd drillpipe-predictive-maintenance pip install -r requirements.txt

Start the FastAPI backend:

uvicorn main:app --reload --port 8000

Start the Streamlit analytics dashboard:

streamlit run app.py

Model artefacts (.pkl files) are not committed to the repository. Train with your labelled coating dataset and place the four artefact files in the root directory before starting the Streamlit app.

This static website (index.html) is hosted via GitHub Pages — no build step required. Enable it under Settings → Pages → Deploy from main branch / root.

Project structure
drillpipe-predictive-maintenance/ ├── main.py # FastAPI routes + WebSocket hub ├── services.py # Waveform, spike detector, RUL, context store ├── rules.py # Integrity scoring & decision logic ├── schemas.py # Pydantic request/response models ├── app.py # Streamlit analytics dashboard ├── index.html # This website (GitHub Pages) ├── requirements.txt └── README.md

Four backend files cover all domain logic. No repository pattern, no service locator, no unnecessary abstraction. Each file has a single clear responsibility: routes, domain services, decision rules, and data schemas.
